### 4. Chord Suggestion Engine (NEW)
Files added:
- `chord_suggestion_engine.js` — pure logic, zero DOM, C++-portable
- `chord_audio.js` — Web Audio playback (triangle oscillators + ADSR)
- `chord_progressions.html` — suggestion panel UI added

**How it works:**
1. User inputs Chord A (root + quality) and Chord B (root + quality)
2. User selects artist setting (15 options)
3. Engine infers key from the two chords (dual-key hypothesis, tests 24 keys)
4. Matches against artist's progressions via degree comparison (stripped Roman numerals)
5. Returns 2-4 VoicedChord objects with artist-specific interval arrays
6. Audio plays: A → suggestions → B via Web Audio API

**VoicedChord output shape:**
```js
{
  symbol: "G7sus4",
  root: 7,               // semitone 0=C
  intervals: [0,5,7,10], // artist-specific voicing from extension_map
  voicing_label: "Dominant 7sus4 — suspended, avoids tritone",
  source: "D'Angelo / Kendrick / Herbie Hancock",
  artist: "dangelo",
  match_type: "direct" | "bridged"
}
```

**Key design:** intervals come from extension_map (artist-attributed), not generic chord symbols. This is what separates it from every other chord plugin.

### 5. Bug Fixed This Session
- matchProgressions was requiring score >= 3 (both chords in same progression) for "direct" match
- In practice, arbitrary input chords rarely appear together in any single artist progression
- Fix: score >= 0.5 (one anchor found) now triggers direct mode — uses found degree as anchor, pulls neighbors
- Result: all 15 artists now return direct matches for common chord pairs
